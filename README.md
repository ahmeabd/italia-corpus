# Italia Corpus

**Hundreds of thousands** of acts as Markdown files  ·  **23** Normattiva collections (full sync) ·  **GBs** of typical working tree (order of magnitude)1  ·  **24h** check cadence  
  
  

1 Counts from a pipeline run that converted every HTML file to Markdown per collection; one collection (*Regi decreti*) hit a disk-full error and is not included—clone size also follows GitHub’s “repo size” badge above.

> The corpus of Italian legislation in Markdown, updated daily, open to everyone.

---

## What this project is

**Italia Corpus Legale** is a public dataset that collects all Italian legislation published on [Normattiva](https://www.normattiva.it) and distributes it as **Markdown**, ready for language models, RAG pipelines, researchers, and anyone who needs structured, machine-readable legal text.

Each act is one file. Each legislative update is a commit.

---

## Why it exists

Italian legislation is technically public, but in practice it is hard to use:

- Proprietary formats or HTML that is difficult to parse
- No ready-to-use dataset for model training
- No open-source Italian legal benchmark
- No AI-friendly version of the law in force

This project addresses all four gaps.

---

## How it is produced

The corpus is generated by a Python script that:

1. Calls Normattiva’s public APIs
2. Downloads predefined collections
3. Converts content to clean Markdown
4. Commits every change
5. Publishes to GitHub automatically

Each run only produces real diffs: if a collection has not changed, no commit is created.

## Contributing

Generic “PRs welcome” does not help much—here is what would move the needle right now:

- **Vector DB smoke tests** — try indexing a subfolder (e.g. *Codici*) with **Qdrant**, **Weaviate**, or **Chroma** and report memory, chunk size, and failure modes (open an issue with timings and versions).
- **Loader sanity checks** — same for **LangChain** / **LlamaIndex** loaders on large trees; document any path or encoding edge cases.
- **Italian legal QA benchmark** — a small, reproducible Q&A set with article-level citations would help compare models and RAG setups.
- **Parser edge cases** — if you spot wrong headings, broken tables, or missing metadata in specific acts, a minimal repro (URL + file path) is gold.

Pull requests are still welcome for parsing fixes, validation scripts, and tooling integrations. Open an issue before structural changes to the on-disk format.

## Contribuire

Un generico “PR benvenute” poco aiuta: ecco cosa servirebbe concretamente adesso:

- **Test su vector DB** — indicizza una sottocartella (es. *Codici*) con **Qdrant**, **Weaviate** o **Chroma** e riporta memoria, dimensione dei chunk e errori (issue con tempi e versioni delle librerie).
- **Loader** — stesso tipo di prova con **LangChain** / **LlamaIndex** su alberi grandi; annota path o encoding “strani”.
- **Benchmark Q&A legale italiano** — un piccolo set riproducibile di domande e risposte con citazione ad articolo servirebbe a confrontare modelli e RAG.
- **Edge case del parser** — titoli sbagliati, tabelle rotte, metadati mancanti: basta un caso minimo (URL Normattiva + percorso file).

Restano benvenute PR su fix di parsing, script di validazione e integrazioni. Apri una issue prima di cambi strutturali al formato su disco.
